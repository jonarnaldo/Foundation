    project schedule and the bank feed. Instead of treating invoices as isolated financial documents, the app treats 
    them as milestones tied to real-world triggers.

    <automated_check_reconciliation>
      Bridges the gap between a physical check deposit and a project milestone by combining
      mobile OCR capture with automated Plaid transaction matching.

      Workflow:
      1. Contractor photographs a physical check on mobile; Veryfi Lens OCR extracts
         check_number, amount, payor_name, and memo_line.
      2. Extracted data is confirmed in a form where the contractor links the check to a
         project and milestone; a pending_checks record is created with status "pending".
      3. Plaid fires a SYNC_UPDATES_AVAILABLE webhook; the backend calls /transactions/sync
         and iterates the "added" array for settled deposits (pending: false).
      4. A two-tier matching engine runs against pending_checks:
         - Tier 1 (high confidence): Plaid check_number equals DB check_number AND amounts match.
         - Tier 2 (fallback): Exact amount match where Plaid posted_date falls within a
           rolling 7-day window of the pending_checks created_at.
      5. On a unique Tier 1 or Tier 2 match: atomically set pending_checks.status →
         "completed", write plaid_transaction_id, set milestones.status → "paid", and
         fire an async user notification.
      6. On a Tier 2 multi-match (ambiguous): set status → "needs_review" and surface
         the check in a manual resolution queue in the UI.

      Key rules:
      - Plaid represents inflows as negative values; matching negates accordingly.
      - A Plaid transaction with pending: true is never used for auto-settlement.
      - Multi-match ambiguity always blocks auto-settlement to prevent incorrect ledger entries.
    </automated_check_reconciliation>

      <top_header>
        - Global search bar (search by check number, project name, or client)
        - Live QuickBooks Sync Indicator badge (far right): Green = connected, Amber = re-authentication needed
      </top_header>

      <main_content>
        - Renders the active page based on sidebar navigation selection
        - Defaults to the Financial Dashboard of the default project, or last opened project if no default is set

        <project_schedule_builder_page>
          - Phase Accordion Cards: Expandable block sections per project phase (e.g., Phase 1: Site Prep, Phase 2: Foundation)
          - Milestone Row Component: Horizontal row per milestone showing title, target dollar value, and Status Badge (Draft, In Progress, Awaiting Inspection, Ready to Invoice, Paid)
          - Drag-and-Drop Grip Handle: Texture dots on the left edge of each milestone row for reordering and phase reassignment
          - "Ready to Invoice" Action Toggle: Slide-toggle that turns green on activation and opens a confirmation modal ("This will generate Invoice #1004 in QuickBooks for $20,000. Proceed?")
          - Inline Media Dock: Thumbnail slot at the end of each milestone row for uploading/viewing inspection PDFs and field photos
          - Plus button (top right): Opens an Add Milestone modal
        </project_schedule_builder_page>

        <project_financial_dashboard_page>
          - Financial Metric Card Grid (four summary cards):
            - Total Contract Value
            - Invoiced to Date
            - Cash Collected
            - Unbilled Progress
          - Cash Flow Burn-Down Chart: Dual-line interactive graph — green line (cumulative income/cleared checks) vs. red line (expenses/subcontractor payouts) over a monthly timeline; hover reveals exact dollar totals per date
          - Dynamic Action-Item Banner: Non-intrusive notification strip surfacing immediate actions (e.g., "$35,000 in approved milestones ready for invoicing")
        </project_financial_dashboard_page>

        <bank_sync_transaction_matcher_page>
          - Left Split-Pane — Unlinked Bank Feed Ledger: Vertical feed of incoming bank transactions showing date, raw bank metadata string, check image thumbnail (if available), and bold deposit amount
          - Right Split-Pane — AI-Suggested Matches: Updates on transaction selection to show likely matching milestones with a Match Score Badge (percentage based on dollar value or memo keyword match)
          - "Link & Sync" Reconciliation Button: Confirms match with animation (transaction slides and locks onto milestone), updates QuickBooks, and sets milestone status to Paid
          - Split-Payment Utility Drawer: Slide-out panel for single checks covering multiple milestones; allows multi-milestone selection with a running balance that decrements to $0
        </bank_sync_transaction_matcher_page>
      </main_content>
    </main_structure>

    <sidebar_left>
      - Collapsible left sidebar with primary navigation
      - Project Dropdown Switcher (top of sidebar): Filter entire app view to a single job site or aggregate portfolio view
      - Navigation links:
        - Project & Schedule Builder
        - Finances
        - Settings
    </sidebar_left>

    <modals_overlays>
      - Add Milestone modal: Triggered by plus button on Project & Schedule Builder; form to define milestone title, dollar value, phase assignment, and completion criteria
      - Ready to Invoice confirmation modal: Confirms QuickBooks invoice generation for a specific milestone (displays invoice number and dollar amount)
      - Transaction detail modal: Opened by clicking a transaction in the bank feed; shows full metadata, check image, and linked milestone
      - Split-payment utility drawer: Slide-out panel for allocating a single check across multiple open milestones
      - Settings modal
    </modals_overlays>
  </ui_layout>

  <key_interactions>
    <global_navigation>
      1. Click the Project Dropdown Switcher (top of left sidebar) to switch the
        entire app view to a different job site, or select "Portfolio View" to
        see aggregated data across all active projects
      2. App defaults to the Financial Dashboard of the user's default project on
        load; if no default is set, it reopens the last active project
      3. Click "Project & Schedule Builder", "Finances", or "Settings" in the
        left sidebar to switch the main content area
      4. Click the collapse arrow on the left sidebar to hide it and expand the
        main content area; click again to restore
      5. Type into the global search bar (top header) to search across check
        numbers, project names, and client names in real-time; click a result
        to navigate directly to that record
      6. Monitor the QuickBooks Sync Indicator badge (top-right header): green
        means connected, amber means re-authentication is required; click the
        amber badge to trigger the QuickBooks OAuth reconnect flow
    </global_navigation>

    <project_schedule_builder>
      1. Click a Phase Accordion Card header to expand or collapse its milestone
        rows; all phases default to expanded on first load
      2. Grab the texture-dot grip handle on the left edge of any milestone row
        and drag vertically to reorder it within its phase; drag across a phase
        boundary to reassign it to a different phase — sort order and phase
        assignment persist immediately on drop
      3. Grab the grip handle on a Phase Accordion Card header to reorder entire
        phases; all child milestones move with their parent phase
      4. Click the plus button (top-right of the page) to open the Add Milestone
        modal; fill in title, dollar value, phase assignment, and completion
        criteria, then click "Save" to append the milestone to the selected phase
      5. Read the Status Badge on each milestone row to understand its current
        state (Draft → In Progress → Awaiting Inspection → Ready to Invoice → Paid);
        badge color updates automatically as criteria are met
      6. Click the inline media dock thumbnail slot at the end of a milestone row
        to upload an inspection PDF or site photo; uploaded files display as
        thumbnail previews; click a thumbnail to open the full document viewer
      7. Flip the "Ready to Invoice" slide-toggle on a milestone row to green;
        a confirmation modal appears showing the projected QuickBooks invoice
        number and dollar amount — click "Proceed" to generate the invoice or
        "Cancel" to revert the toggle
      8. Click directly on a milestone title, dollar value, or criteria text to
        edit it inline; changes save on blur or Enter keypress
    </project_schedule_builder>

    <project_financial_dashboard>
      1. Read the four Financial Metric Cards at a glance (Total Contract Value,
        Invoiced to Date, Cash Collected, Unbilled Progress); cards update in
        real-time as transactions are logged and milestones are reconciled
      2. Hover over any point on the Cash Flow Burn-Down Chart to reveal a tooltip
        showing the exact cumulative income (green line) and cumulative expense
        (red line) dollar totals for that date
      3. Use the date range selector above the chart to switch between monthly,
        quarterly, and project-lifetime views
      4. Read the Dynamic Action-Item Banner below the metric cards for proactive
        alerts (e.g., "3 approved milestones totaling $35,000 are ready to invoice");
        click the banner CTA to jump directly to the relevant milestones in the
        Schedule Builder
      5. Click "Dismiss" on an individual alert in the banner to suppress it
        without taking action
      6. Click "Export Report" and select PDF or CSV to generate a financial
        report; choose a recipient label (Lender, Client, Accountant) before
        confirming — the file is generated and downloaded immediately
      7. Toggle between "This Project" and "Portfolio View" (top of dashboard)
        to switch between individual project drill-down and the aggregate
        multi-project overview
    </project_financial_dashboard>

    <bank_sync_transaction_matcher>
      1. The Left Split-Pane auto-populates with cleared deposits from connected
        bank accounts via Plaid on page load; new transactions appear at the top
        sorted by posted date
      2. Click any transaction row in the Left Split-Pane to select it; the Right
        Split-Pane immediately updates to display AI-suggested milestone matches
        ranked by Match Score Badge (percentage reflecting dollar-amount proximity
        and OCR memo keyword overlap)
      3. Click a suggested match in the Right Split-Pane to highlight it; review
        the match score rationale (amount match, date proximity, keyword hits)
        before confirming
      4. Click "Link & Sync" to confirm the match; the transaction row animates
        sliding across and locking onto the milestone, the milestone status
        updates to Paid, and a QuickBooks write-back fires to auto-generate the
        itemized invoice and mark it as Paid
      5. Click a transaction's check image thumbnail to open the Transaction Detail
        Modal, which displays full bank metadata, the enlarged check image, OCR-
        parsed text, and any currently linked milestone
      6. Click "Ignore" on a transaction row to exclude it from the unmatched
        feed; click "Unignore" from the audit log view to restore it
      7. Click "Split Payment" on a transaction row to open the Split-Payment
        Utility Drawer; select multiple open milestones from the list and enter
        an allocated amount for each — a running balance decrements in real-time
        toward $0; click "Confirm Split" when the balance is fully allocated
      8. Click "Undo Match" on a previously reconciled transaction to unlink it
        from its milestone, revert the milestone status, and queue a QuickBooks
        reversal — all steps are logged to the audit trail
      9. Click "View Audit Log" (bottom of the page) to open the full historical
        reconciliation trail showing every match, confirmation, split, and
        QuickBooks sync event with actor identity and timestamp
    </bank_sync_transaction_matcher>

    <modals_and_overlays>
      1. Add Milestone Modal: Opened via the plus button on the Schedule Builder;
        complete all required fields (title, dollar value, phase, criteria) and
        click "Add Milestone" to append; click outside the modal or press Escape
        to dismiss without saving
      2. Ready to Invoice Confirmation Modal: Triggered by flipping the
        "Ready to Invoice" toggle; displays the projected QuickBooks invoice
        number and exact dollar amount for review; click "Proceed" to commit
        or "Cancel" to revert — cannot be dismissed by clicking outside to
        prevent accidental invoice generation
      3. Transaction Detail Modal: Opened by clicking a transaction row in the
        bank feed; displays full metadata, check image, OCR text, and linked
        milestone; click "Open in Schedule Builder" to navigate directly to
        the matched milestone; close via the X button or Escape
      4. Split-Payment Utility Drawer: Slides in from the right edge of the
        screen over the transaction matcher; supports multi-milestone selection
        with per-row amount inputs and a live running balance counter; closing
        the drawer without confirming discards all unsaved allocations
      5. Settings Modal: Opened via the "Settings" sidebar link; organized into
        tabbed sections (Account, Integrations, Notifications); changes save
        per-section via individual "Save" buttons
    </modals_and_overlays>

    <quickbooks_integration>
      1. Click "Connect QuickBooks" in Settings → Integrations to initiate the
        OAuth flow; complete authentication in the QuickBooks popup window;
        on success the top-header badge turns green
      2. Click the amber QuickBooks Sync Indicator badge at any time to trigger
        re-authentication without navigating away from the current page
      3. Assign a QuickBooks item code to any milestone or line item via the
        inline item-code selector on the Schedule Builder; codes are pulled
        live from the connected QuickBooks account
      4. QuickBooks invoices are auto-generated and marked as Paid when "Link &
        Sync" is confirmed in the transaction matcher — no manual QB entry required
      5. Click "Force Sync" in Settings → Integrations to manually trigger a
        write-back for any records showing a pending or failed QuickBooks sync status
    </quickbooks_integration>

    <plaid_bank_connection>
      1. Click "Connect Bank Account" in Settings → Integrations to launch the
        Plaid Link modal; search for and select the bank institution, then
        authenticate with online banking credentials inside the Plaid-hosted flow
      2. On successful connection, the bank account appears in the connected
        accounts list with its last-synced timestamp
      3. Transactions begin populating the bank feed automatically; click
        "Sync Now" next to any connected account to force an immediate Plaid pull
        outside the standard refresh cadence
      4. Click "Remove" next to a connected account to disconnect it; existing
        matched transactions are retained in the audit log but no new transactions
        will be pulled
    </plaid_bank_connection>
  </key_interactions>
